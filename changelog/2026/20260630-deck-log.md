# V-Sekai Multiplayer Fabric Deck Log — 2026-06-30 (Tuesday)

Date: 2026-06-30

Made the loot-action slice runnable on a bare Windows 11 machine, with no Docker and no WSL. The slice
now ships as a downloadable release: the authoritative server, the client, and the bots that fill the
four-player party are native Windows executables, persistence is in-process SQLite, and an optional
native observability stack (VictoriaMetrics + OpenTelemetry) runs alongside. One double-click on
`run-demo.cmd` starts the whole thing.

Along the way the loop closed its last gap — the loot a player wins now travels back to the client and
shows in the hub — and the Windows build and packaging were hardened across several release cuts,
`v0.1.0-dev.3` through `v0.1.0-dev.6`.

## Added

- A native Windows demo of the [godot-loop-slice](https://github.com/v-sekai-multiplayer-fabric/godot-loop-slice)
  vertical slice, on the [v0.1.0-dev.6 release](https://github.com/v-sekai-multiplayer-fabric/godot-loop-slice/releases/tag/v0.1.0-dev.6):
  a plain-exe `…-demo-windows.zip` plus three unsigned MSI installers — the all-in-one demo, a
  server-only install (binds all interfaces and prints the host's LAN address), and a client-only
  install (connects to a given server address).
- Double-click `.cmd` launchers (`run-demo.cmd`, `run-server.cmd`, `run-client.cmd`) that run the
  PowerShell launcher as an in-memory scriptblock, so Windows execution policy and Mark-of-the-Web
  never block a downloaded script — no scoop and no policy change. The MSI Start-menu shortcuts point
  at the same launchers.
- The loop's persistence round trip now closes on the client: the granted item is synced back and
  shown in the hub on return, and the playable-loop smoke asserts the winning client's inventory
  matches the committed profile row.

## Changed

- Windows distribution moved from a signed MSIX to an unsigned MSI. MSIX must be signed to install and
  the self-signed cert step failed on the runner, whereas an MSI installs through `msiexec` without a
  signature or admin. WiX is pinned to v5 because v6 and later require accepting the paid OSMF EULA.

## Fixed

- The Windows release build failed because the runner's backslash stage path (`D:\a\_temp\…`) reached
  an Elixir `Path.wildcard` glob, where a backslash is a glob escape, so the exported editor was never
  found. The stage path is normalized to forward slashes, which also fixes the same latent bug in the
  MSIX build script.

## Removed

- The MSIX release workflow, superseded by the unsigned MSI path.
